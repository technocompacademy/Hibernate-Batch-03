Association Mapping
===================
--> Association Mapping refers to mapping relationships between Java entity classes based on their real-world relationships.

--> These relationships are reflected in the database using foreign keys and are managed in Hibernate using special annotations.  

@Id 			: used to represnt Primary Key.
@OneToOne  	: Maps a one-to-one association 
@OneToMany  : Maps a one-to-many association 
@ManyToOne  : Maps a many-to-one association 
@ManyToMany : Maps a many-to-many association 

@JoinColumn : Specifies the foreign key column 
@JoinTable  : Used in many-to-many to define the association table 

Q. Why is Association Mapping Needed? 
-------------------------------------
--> When two entities have a relationship (One-to-Many, Many-to-One, One-to-One, Many-to-Many) in the relational model, we need Association Mapping to provide a connection between the Object Model (Entity classes) and the Relational Model (Database tables). 

--> Association Mapping is used to manage and map foreign key relationships from the relational model into the object-oriented model. 

--> Because of Association Mapping, a Java developer does not need to manually write SQL JOINs. Hibernate automatically handles joining associated entities behind the scenes.


Types of Associations 
----------------------- 
1. One-to-One  : One object is mapped to exactly one object  
   Example: One Car has one Engine 

2. One-to-Many : One object is mapped to multiple objects  
   Example: One Department has Many Employees  

3. Many-to-One : Many objects are mapped to one object  
   Example:  Many Employees are having One Department 

4. Many-to-Many : Many objects are mapped to many objects  
   Example: Many Students have enrolled Many Courses.


Note:
------
--> Relationship/Association can be Unidirectional or Bi-directional.

Customar    ---[1:M]----  Order    ---> Unidirectional
Department  ---[1:M]----  Employee  --> Bi-directional
